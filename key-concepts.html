Key Concepts

This page covers seven key concepts that are important to understand when learning about web servers. Knowing these terms makes it a lot easier to follow along with how the web actually works behind the scenes.

1. HTTP (HyperText Transfer Protocol)

HTTP is the protocol that browsers and web servers use to communicate with each other. When you visit a website, your browser sends an HTTP request and the server sends back an HTTP response with the content you asked for. HTTPS is just the secure version of HTTP that encrypts the data so it cannot be intercepted.

2. Client-Server Model

The client-server model is the core architecture the web is built on. The client, usually a web browser, sends a request for content, and the server sends back whatever was asked for. This model has been around since the very first web server at CERN and is still the standard today.

3. Static Web Server

A static web server just stores files and sends them as-is when someone requests them. There is no processing happening on the server side. Static servers are fast, easy to set up, and work well for sites that do not need user accounts or live data, like portfolios or documentation pages.

4. Dynamic Web Server

A dynamic web server builds content on the fly based on what the user is asking for. It works with an application server and a database to put together a personalized response. Social media, shopping sites, and anything with a login system all need dynamic web servers to function.

5. URL (Uniform Resource Locator)

A URL is just the address for a resource on the web. When you type one into a browser, it uses that address to figure out which server to contact and what specific file to ask for. A typical URL includes a protocol like http or https, a domain name, and a path to the specific file.

6. Reverse Proxy

A reverse proxy is a server that sits in front of other web servers and passes client requests to them. It is used to improve performance, add a layer of security, and make scaling easier. Nginx is a good example of a server that is often used as a reverse proxy to route traffic to the right backend server.

7. Load Balancer

A load balancer spreads incoming traffic across multiple servers so no single one gets overwhelmed. This helps keep sites reliable and prevents crashes during periods of high traffic. Big platforms like social media sites and online stores rely on load balancing to handle millions of requests every day.
